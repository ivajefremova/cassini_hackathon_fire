01 · Data Pipeline

How the satellite data flows

Five steps from raw Sentinel-2 imagery to a contamination risk score, running on Copernicus cloud infrastructure.

STEP 01
Sentinel-2 L2A Load
↳ openEO · CDSE
Pre-fire (Jul–Sep) and post-fire (Oct) median composites. Bands B08 (NIR) + B12 (SWIR). Max 85% cloud cover.
STEP 02
dNBR Computation
↳ openEO · Cloud
NBR = (B08–B12)/(B08+B12). dNBR = NBR_pre − NBR_post. Higher value → more severe burn.
STEP 03
Catchment Extraction
↳ rasterio · DEM
Per-lake catchment bbox masks dNBR raster. Pixel count × area → burned hectares by severity class.
STEP 04
CORINE Land Use
↳ EEA · Static
Fixed land-use fractions (forest, agricultural, urban) determine which contaminants are emitted.
STEP 05
Forecast Engine
↳ FastAPI · Python
Emission factors × severity multiplier × exponential decay → kg/month per contaminant per lake.
sentinel_pipeline.py · dNBR computation
# ── Pre-fire composite ──────────────────────────────────────── cube_before = conn.load_collection( "SENTINEL2_L2A", spatial_extent=SPATIAL_EXTENT, temporal_extent=BEFORE_DATES, # "2024-07-01" → "2024-09-20" bands=["B08", "B12"], max_cloud_cover=85, ) nir_before = cube_before.band("B08") swir_before = cube_before.band("B12") nbr_before = (nir_before - swir_before) / (nir_before + swir_before) nbr_before_mean = nbr_before.reduce_dimension(dimension="t", reducer="mean") # ── Post-fire composite ─────────────────────────────────────── cube_after = conn.load_collection( "SENTINEL2_L2A", spatial_extent=SPATIAL_EXTENT, temporal_extent=AFTER_DATES, # "2024-10-01" → "2024-10-25" bands=["B08", "B12"], max_cloud_cover=85, ) nir_after = cube_after.band("B08") swir_after = cube_after.band("B12") nbr_after = (nir_after - swir_after) / (nir_after + swir_after) nbr_after_mean = nbr_after.reduce_dimension(dimension="t", reducer="mean") # ── dNBR = pre – post (positive = burned) ─────────────────── dnbr = nbr_before_mean - nbr_after_mean result = dnbr.save_result(format="GTiff") result.download("output/dnbr_corinthia.tif")
